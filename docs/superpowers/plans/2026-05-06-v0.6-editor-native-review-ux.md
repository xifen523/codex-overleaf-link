# v0.6 Editor-Native Review UX Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement hunk-level review, editor jump, large-diff safeguards, scoped shortcuts, and v0.6 release metadata without changing the v0.5 writeback safety model.

**Architecture:** Keep native sync output backward-compatible and implement hunk review as a content-layer model over existing `patches`. Page bridge gains a focused `jumpToPosition` method; actual writes remain existing `applyOperations` patch writes with stale guards.

**Tech Stack:** Chrome MV3 content scripts, plain CommonJS/UMD JavaScript, Node.js built-in test runner, zero runtime dependencies.

---
