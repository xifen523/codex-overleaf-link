# v0.4 Distribution and Update Foundation Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Advance the released v0.3.0 codebase to an initial v0.4.0 release candidate focused on compatibility diagnostics, release artifacts, installer/update guidance, and Chrome Web Store preparation.

**Architecture:** Add a small shared compatibility module consumed by content scripts, popup, background service worker, and tests. Gate native mutation/execution methods before they reach the native host, extend `bridge.ping` metadata, and keep release tooling separate under `scripts/` so runtime behavior does not depend on packaging code.

**Tech Stack:** Chrome MV3 extension, Node.js CommonJS runtime for native host and tests, Node ESM scripts for installer/release tooling, GitHub Actions.

---

## File Responsibility Map

- `extension/src/shared/compatibility.js`: shared version/protocol/capability classifier and native method policy; must export `CodexOverleafCompatibility` in browsers and `module.exports` in tests.
- `native-host/src/taskRunner.js`: native `bridge.ping` metadata provider.
- `extension/src/background.js`: service worker native request policy and compatibility blocking before background-to-native forwarding.
- `extension/src/contentScript.js`: panel diagnostics, run gate user feedback, and compatibility-aware native calls.
- `extension/src/popup.js` and `extension/popup.html`: popup connection status and compatibility guidance.
- `extension/src/content/otWarmMirrorController.js`: compatibility-aware `mirror.patchFiles` dispatch for OT warm mirror.
- `scripts/verify-release.mjs`: release metadata, changelog, Web Store docs, and version consistency checks.
- `scripts/build-release.mjs`: release artifact packaging, release notes extraction, manifest, and checksums.
- `.github/workflows/release.yml`: tag-based release artifact publishing.
- `docs/chrome-web-store/*.md`: Web Store permission, privacy, listing, and checklist preparation docs.
